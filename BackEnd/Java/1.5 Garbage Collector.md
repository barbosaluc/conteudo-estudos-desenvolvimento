Mecanismo automático de gerenciamento de memoria.
### Como funciona?

No Java, quando você cria um novo objeto, a memória para esse objeto é alocada na **Heap**. Em linguagens como C e C++, o programador é responsável por liberar essa memória quando o objeto não é mais necessário. Se ele se esquecer de fazer isso, pode ocorrer um "vazamento de memória" (memory leak).

O Garbage Collector resolve esse problema de forma automática:

1. <font color="#ff0000">Identificação de Objetos Inalcançáveis:</font> O GC rastreia quais objetos ainda estão sendo referenciados por alguma parte do seu programa. Um objeto que não tem mais nenhuma referência apontando para ele é considerado "inalcançável" (ou lixo).
2. <font color="#ff0000">Liberação de Memória:</font> Quando o GC é acionado, ele "coleta" esses objetos inalcançáveis e libera a memória que eles ocupavam, tornando-a disponível para a criação de novos objetos.

---

### Quando o Garbage Collector é executado?

A JVM decide o momento ideal para executar o GC, geralmente quando a memória da **Heap** está ficando escassa. O processo de coleta é chamado de **"stop-the-world"**, pois toda a execução do programa é temporariamente pausada para que o GC possa fazer seu trabalho, garantindo que o estado da memória não mude durante a coleta.

As versões modernas da JVM, no entanto, usam algoritmos de GC mais avançados que minimizam essas pausas, permitindo que as aplicações rodem de forma mais fluida.
