É um código intermediário gerado pelo compilador, interpretado pela JVM.
Possui um conjunto de instruções de baixo nível, mas que não é o código de máquina nativo.
### como se encaixa no ecossistema Java

1. <font color="#ff0000">Código-Fonte:</font> Você escreve seu programa em arquivos de código-fonte (`.java`).
2. **<font color="#ff0000">Compilação</font>:** O **Java Compiler (`javac`)** transforma o seu código-fonte em um arquivo de bytecode (`.class`).
3. **<font color="#ff0000">Execução</font>:** A JVM pega o arquivo `.class` (o bytecode) e o executa.

A grande vantagem do bytecode é a **portabilidade**. Como o bytecode é genérico e não depende de nenhuma arquitetura de hardware, o mesmo arquivo `.class` pode ser executado em qualquer sistema operacional que tenha uma JVM instalada. A JVM é a responsável por traduzir o bytecode para o código de máquina nativo do sistema (seja Windows, Linux, macOS, etc.) em tempo de execução.
